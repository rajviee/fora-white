1. Core Concepts - Self vs Team Tasks

CRITICAL CONCEPT: SELF TASK vs TEAM TASK LOGIC

The application has a FUNDAMENTAL distinction between "Self Tasks" and "Team Tasks". This isSelfTask flag controls task visibility, filtering, permissions, and reporting throughout the entire system.

WHAT IS A SELF TASK? (isSelfTask=true)

A task where:

  • Exactly ONE assignee AND exactly ONE observer
  • Both are the SAME person (the task creator)
  • Automatically calculated at creation:
const selfTask = assignees.length === 1 && observers.length === 1 &&
  assignees[0] === req.user.id && observers[0] === req.user.id;
task.isSelfTask = selfTask;

Example: Admin creates "Buy groceries" and assigns it only to themselves (they are both the only assignee and only observer)

WHAT IS A TEAM TASK? (isSelfTask=false)

A task where:

  • Multiple assignees, OR
  • Multiple observers, OR
  • Assignee â‰  observer (different people)

Example: Admin creates "Complete Q1 report" assigned to 3 employees with Admin as the observer

WHY THIS DISTINCTION EXISTS

  1. PRIVACY: Self tasks are completely private to the creator. No one else sees them.
  1. ORGANIZATION: Separates personal tasks from team collaboration work
  1. REPORTING: Completely separate report views for self vs team tasks
  1. FILTERING: Users toggle between viewing their personal tasks vs team tasks

BACKEND LOGIC IMPLEMENTATION

Task Creation

// taskController.createTask
// Automatically calculates isSelfTask based on assignees/observers
// Stores boolean flag in task document

Task Retrieval

// taskController.getTaskList
// Frontend sends query parameter: ?isSelfTask=true OR ?isSelfTask=false
// Backend filters completely differently based on this flag

For SELF TASKS (isSelfTask=true)

Database Query: {
  isSelfTask: true,
  createdBy: userId,      // ONLY creator can see their self tasks
  company: companyId
}
Result: User sees ONLY their own personal tasks

For TEAM TASKS (isSelfTask=false)

If user is ADMIN:

Database Query: {
  isSelfTask: false,
  company: companyId    // Admin sees ALL company team tasks
}
Result: Admin sees all team tasks in the company

If user is EMPLOYEE/SUPERVISOR:

Database Query: {
  isSelfTask: false,
  company: companyId,
  $or: [
    { assignees: userId },    // Tasks assigned to them
    { observers: userId },    // Tasks they're observing
    { createdBy: userId }     // Tasks they created
  ]
}
Result: Employee sees only team tasks they're involved in

FRONTEND BEHAVIOR

  • Dashboard and Task screens have "Self" vs "Team" toggle/tabs
  • Clicking "Self" sends API request with ?isSelfTask=true
  • Clicking "Team" sends API request with ?isSelfTask=false
  • User sees COMPLETELY DIFFERENT task lists based on this flag
  • The two lists never overlap

PERMISSION RULES

SELF TASKS:

  • Creator: Full access (view, edit, delete, complete)
  • Everyone else: CANNOT see the task at all (total privacy)

TEAM TASKS:

  • Admin: Can view, edit, delete ALL company team tasks
  • Observer: Can view task, approve/reject task completion
  • Assignee: Can view, edit (limited fields), mark as complete (triggers approval)
  • Creator: Can view, cannot edit unless they are Observer
  • Other users: Cannot see tasks they're not part of
  • If a user is both Assignee and Observer: Observer role is considered

TASK COMPLETION WORKFLOW

SELF TASK Completion:

  1. Creator marks task as complete
  1. Status IMMEDIATELY becomes "Completed" (no approval needed)

TEAM TASK Completion (by Assignee):

  1. Assignee marks task as complete
  1. Status becomes "Completed" for assignee but "For Approval" for observer/admin
  1. If user is both assignee and observer, status shows "For Approval"
  1. Notification sent to ALL observers
  1. Observer approves â†’ Status becomes "Completed"
  1. Observer rejects â†’ Status returns to Pending state

TEAM TASK Completion (by Observer or Admin):

  1. Observer/Admin marks task as complete
  1. Status IMMEDIATELY becomes "Completed" (no approval needed)

REPORTING & STATISTICS

The system maintains separate reporting for self and team tasks:

Admin Report (/reports/admin-report-summary)

  • Shows ONLY Team Tasks (isSelfTask=false)
  • Aggregates data across all employees
  • Used for company-wide performance metrics

Self Report (/reports/self-report-summary)

  • Shows ONLY Self Tasks (isSelfTask=true)
  • Shows only the logged-in user's personal tasks
  • Used for personal task tracking

Dashboard Statistics

  • Calculated separately for Self and Team tasks
  • User toggles between "Self" and "Team" views
  • Charts and graphs update based on selected view

IMPORTANT EDGE CASES

  1. Admin creating task for only themselves = Self Task (even though they're admin, if assignees[0] === observers[0] === adminId)
  1. Editing a Team Task and removing all assignees except yourself does NOT automatically convert it to a Self Task. The isSelfTask flag is generally set at creation and doesn't auto-update.
  1. Self Tasks do NOT appear in:
    • Admin's team task view or list
    • Admin Report (company-wide reports)
    • Company statistics for team tasks
    • Other users' task lists or searches
  1. You CANNOT convert a Self Task to Team Task or vice versa after creation (the isSelfTask flag is immutable in most controllers)
  1. When searching or filtering tasks, the isSelfTask parameter MUST be specified or you'll get validation errors from the backend
