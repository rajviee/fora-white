State Management

Zustand Store (useUserStore.js)

// Global state for authentication
{
  user: null | UserObject,        // Current logged-in user
  isLoading: boolean,             // Loading state
  error: string | null,           // Error messages
  isInitialized: boolean,         // App initialization status
  
  // Actions
  initializeStore: async () => {},  // Load stored auth data
  login: async (credentials) => {}, // Login user
  logout: async () => {},           // Logout user
  fetchUser: async () => {},        // Refresh user data
  isAuthenticated: () => boolean,   // Check auth status
  clearError: () => {}              // Clear errors
}

React Query (TanStack Query)

  • Used for server state management
  • Automatic caching, refetching, and background updates
  • Example:
const { data, isLoading } = useQuery({
  queryKey: ['tasks'],
  queryFn: () => api.get('/task/getTaskList')
});

Context API

  • DashboardContext: Shares dashboard data across components
  • RouteContext: Tracks current route for navigation highlighting

Routing Structure

Deep linking configuration:

/login → LoginScreen
/dashboard → Dashboard
/task → TaskList
/task/add → AddTask
/task/:id → TaskDetails
/employees → EmployeesList
/employees/add → AddEmployee
/employees/:id → EmployeeDetails
/reports → ReportsList
/reports/admin-report → AdminReport
/reports/employee-report → EmployeeReport
/notification → NotificationPage
/discussion → Discussion
/settings → Settings
/profile → Profile

Key UI Components

  • TaskCard: Displays task summary with status, priority, assignees
  • UserAvatar: Shows user profile picture with fallback initials
  • CustomButton: Styled button component with loading states
  • DateTimePicker: Cross-platform date/time selection
  • DropdownPicker: Select dropdown for priorities, statuses
  • NotificationCard: Displays notification with action buttons
  • StatCard: Dashboard statistics display
  • ChartComponents: Various charts (pie, bar, line)

Mobile vs Web

const { width } = useWindowDimensions();

{isMobile ? <MobileView /> : <WebView />}

Development Setup

# Navigate to frontend directory
cd foratask-frontend

# Install dependencies
npm install

# Configure environment variables
# Create .env file with:
EXPO_PUBLIC_API_URL=http://192.168.0.101:3000
EXPO_PUBLIC_ENCRYPT_KEY=forataskencryptkey
EXPO_PUBLIC_SIGN_KEY=foratasksignkey

# Start Expo development server
npm start

# Run on web
npm run web

# Run on Android
npm run android

# Run on iOS
npm run ios
